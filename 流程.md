# 流程

## 整体

动机\-\-调研（已有此部分）发现所有防御在单一节点，想做全链；

考虑到适配性，而且不同的agent框架记忆的形成路径不同？因此需要分类进行。

**实际的分类方式：按"谁触发写入"来分类**，而不是按框架名称分类。因为同一框架（比如 LangChain）可以配置成完全不同的写入模式，框架名字不是核心变量，写入触发机制才是。

根据记忆的形成路径，找每个节点可能有的攻击方式。

思路：每个节点的攻击防御\-\-考虑组合起来，是否1\+1\>=2\-\-每个类解决后\-\-再次组合，是否有重复的地方，是否1\+1\>=2

初期一个类一个人解决，单独建文件，可并行。

大类：

### 类型一：框架自动写入（Automatic Write）

代表：LangChain ConversationBufferMemory、AutoGen 对话日志

```Plain Text
用户输入
    ↓
Agent 生成回复
    ↓
框架自动调用 save_context()
    ↓
内容原文写入存储（无过滤，无判断）
```

触发者：**框架本身**，每轮对话无条件执行。Agent 对这个写入是无感知的。

**每个节点的威胁：**

这是最高风险的路径类型。攻击者只需要能和 Agent 对话，就能稳定地向记忆中写入任意内容。MemoryGraft 的攻击正是针对这类路径——不需要白盒，不需要权限，直接写。

### 类型二：Agent 自主写入（Agent-Initiated Write）

代表：MemGPT/Letta（archival\_memory\_insert）、ReAct \+ 记忆工具、AutoGPT（memory\_add 命令）

```Plain Text
外部内容进入上下文（用户输入 / 工具返回值 / 网页内容）
    ↓
Agent 推理："这个值得记住吗？"
    ↓
Agent 决定调用 memory_write 工具
    ↓
框架执行写入（完全信任 Agent 的决定）
```

触发者：**Agent 自身的推理判断**。

**每个节点的威胁：**

这是 **MINJA 攻击的精确攻击面**。攻击者不需要写入权限——他操控的是 Agent 的"值得记住"判断，让 Agent 替他完成写入。

---

### 类型三：反思合成写入（Reflection/Synthesis Write）

代表：LangChain ConversationSummaryMemory、OpenAI Dreaming V3、CrewAI 实体抽取

```Plain Text
原始对话内容
    ↓
二次 LLM 处理（摘要/反思/实体提取）
    ↓
生成的"蒸馏事实"写入存储
    ↓
被当作可信知识注入未来上下文
```

触发者：**后台 LLM 进程**，通常异步执行，用户无感知。

**每个节点的威胁：**

这是最隐蔽的注入路径。Tenable TRA\-2025\-11 证实了这个漏洞在 ChatGPT Dreaming 架构中实际存在：恶意网页内容被异步 Dreaming 进程合成为持久记忆，在未来对话中以"你之前记住的事实"的形式出现。

---

### 类型四：外部批量写入（External/Developer Write）

**代表**：OpenAI Assistants 文件上传、RAG 知识库预填充、企业知识库

## 代码块
1. 开发者 / 外部系统上传文档  
2. ↓  
3. 分块 → 嵌入 → 存入向量数据库  
4. ↓  
5. Agent 查询时检索注入上下文  

**触发者**：开发者或外部数据管道，不经过 Agent 推理。

## 每个节点的威胁

| 节点 | 威胁 | 原因 |
|------|------|------|
| 文档上传阶段 | 供应链投毒（PoisonedRAG） | 攻击者影响上传的文档内容；仅需注入约 5 条精心构造的文档即可达到 90% 攻击成功率 |
| 嵌入/分块阶段 | 语义混淆注入 | 恶意指令被拆分为多个 chunk，每个 chunk 单独看无意义，组合检索后形成有效攻击 |
| 检索阶段 | 触发词检索劫持（AgentPoison） | 优化触发词使恶意文档在特定查询下总是排名第一 |

---

### 类型五：多 Agent 共享写入（Multi-Agent Shared Write）

代表：AutoGen 共享对话日志、CrewAI crew memory、多 Agent 系统共享知识库

```Plain Text
Agent A 处理外部不可信内容（网页、工具返回）
    ↓
Agent A 的输出写入共享记忆
    ↓
Agent B、C... 读取共享记忆，信任级别等同于自身输出
```

触发者：**任意 Agent**，写入权限完全平等（扁平信任模型）。

#### 构建测试数据集

每类最少20个（共80\+），加上对应良性样本40\+。先从最小可用规模开始，不要追求完美。有现成的最好（需要你调研看看），权威数据集可以直接拿来用。如果没有，才需要自己构造，可以借助论文（research一下）中的构造方式。例如：Prompt Injection样本可以从 [JailbreakBench](https://github.com/JailbreakBench/jailbreakbench) 和 [Ignore Previous Prompt](https://github.com/agencyenterprise/promptinject) 直接下载，筛选适合Agent场景的，20条起步；Tool Misuse样本：在你的系统里手工构造5\-10个"越权工具调用"场景，截图\+记录输入输出；Memory Poisoning / Agent Hijacking：各手工构造10条。

### 消融评估实验，回答"每个节点是否防住了对应攻击阶段"以及"组合是否 1\+1\>2"。

指标可以有：

- `Prec`（Precision）：被判为攻击的样本里，真正是攻击的比例。= TP/\(TP\+FP\)，衡量误报

- `Rec`（Recall）：所有攻击样本里，被成功拦截的比例。= TP/\(TP\+FN\)，衡量漏报

- `F1`：Precision 和 Recall 的调和平均，综合指标

- `FPR`（False Positive Rate）：正常样本里被误判为攻击的比例。= FP/\(FP\+TN\)，衡量误伤

- `P50/P95/P99ms`：延迟百分位（只在 ALL 行显示），单位毫秒
